In multiple dimensions, the gradient is the vector of (partial derivatives)
along each dimension

Numerical gradient: approximate, slow, easy to write
current W -> W + h -> gradient dW

This is silly. The loss is just a function of W
Calculus! Use calculus to compute an analytic gradient

Analytic gradient: exact, fast, error-prone
current W -> gradient dW

In practice: Always use analytic gradient, but check implementation with numerical gradient.
This is called a gradient check. (using as debugging tool)

Backpropagation
f(x,y,z) = (x+y)z
x= -2, y= 5, z= -4
q = x + y, dq/dx=1, dq/dy=1
f = qz, df/dq=z, df/dz=q
want: df/dx, df/dy, df/dz
df/df=1
df/dz=q=3
df/dq=z=-4
df/dx=df/dq*dq/dx=-4*1=-4
df/dy=df/dq*dq/dy=-4*1=-4
chain rule

x(dL/dx), y(dL/dy) -> f(dz/dx, dz/dy) -> z
                                      <- dL/dz
dz/dx, dz/dy: "local gradient" 

sigmoid function : 1/(1+e^-x)
sigmoid gate : *-1 -> exp -> +1 -> 1/x

patterns in backward flow
add gate: gradient distributor(동등하게 나눠줌)
max gate: gradient router(한쪽에 몰빵함)
mul gate: gradient switcher(합이 0이 됨)

Gradients add at branches

Gradients for vectorized code
x, y, z are now vectors

dz/dx : This is now the Jacobian matrix(야코비 행렬)
(derivative of each element of z with respect to each element of x)

Vectorized operations
4096-d input vector -> f(x)=max(0,x) (elementwise) -> 4096-d output vector

class MultiplyGate(object):
	def forward(x, y):
		z = x*y
		self.x = x # must keep these around!
		self.y = y
		return z
	def backward(dz):
		dx = self.y * dz # [dL/dz*dz/dx]
		dy = self.x * dz # [dL/dz*dz/dy]
		return [dx, dy]

Fully Connected Layer
32x32x3 image -> stretch to 3072 x 1

imput x -> Wx -> activation
3072x1 -> 10x3072, 3072x1 -> 10x1
1 number: the result of taking a dot product between a row of W

Convolutional Layer
32x32x3 image -> preserve spatial structure
heightxwidthxdepth

5x5x3 filter
Convolve the filter with the image
i.e "slide over the image spatially, computing dot products"

Filters always extend the full depth of the input volume

1 number:
the result of taking a dot product between the filter
and a small 5x5x3 chunk of the image
(i.e. 5*5*3 = 75-dimensional dot product + bias)

convolve(slide) over all spatial locations
-> activation map
32x32x3 -> 5x5x3 -> 28x28x1

consider a secon filter -> multiple filters -> activation maps
For example, if we had 6 5x5 filters, we'll get 5 seperate activation maps:
we stack these up to get a "new image" of size 28x28x6!

Preview: ConvNet is a sequence of Convolution Layers, intespersed with activation functions
32x32x3 -> CONV, RELU -> 28x28x6 -> CONV, RELU -> 24x24x10 -> CONV, RELU -> ...

low-level features -> mid-level features -> high-level features -> linearly seperable classifier ->
hierarchy

Zeiler and Fergus, 2013

one filter -> one activation map

we call the layer convolutional because it is related to convolution of two signals
elementwise multiplication and sum of a filter and the signal(image)

Input -> CONV, RELU ... -> ... POOL -> FC

A closer look at spatial dimensions:
32x32x3 image, 5x5x3 filter
-> convolve(slide) over all spatial locations
28x28x1 activation map

7x7 input (spatially) assume 3x3 filter -> 5x5 output
(with stride 2) -> 3x3 output
(with stride 3) -> doesn't fit!
cannot apply 3x3 filter on 7x7 input with stride 3

input size : N
filter size : F
Output Size: (N-F)/stride+1
e.g. N = 7, F = 3:
stride 1: (7-3)/1+1=5
stride 2: (7-3)/2+1=3
stride 3: (7-3)/3+1=2.33 (x)

In practice: Common to zero pad the border
e.g. 7x7 input, 3x3 filter, stride 1
pad with 1 pixel border -> what is the output?
7x7 output!

in general, common to see CONV layers with stride 1,
filters of size FxF, and zero-padding with (F-1)/2.
(will preserve size spatially)
F = 3 : zero pad with 1
F = 5 : zero pad with 2
F = 7 : zero pad with 3

Summary.
To Summarize the Conv Layer
volume W1xH1xD1
Requires 4 hyperparameters:
1) # filters K,
2) Filter size F,
3) stride S, 
4) zero padding P
Produces a volume of size W2xH2xD2
W2 = (W1-F+2P)/S+1
H2 = (H1-F+2P)/S+1
D2 = K

Common settings:
K = (powers of 2, e.g. 32, 64, 128, 512)
- F=3, S=1, P=1
- F=5, S=1, P=2
- F=5, S=2, P=?
- F=1, S=1, P=0

btw, 1x1 convolution layer make perfect sense

56x56x64 -> 32 filters 1x1 CONV ->56x56x32
each filter has size 1x1x64, and performs a 64-dimensional dot product

The brain/neuron view of CONV Layer

32x32x3 image -> 5x5x3 filter -> 1 number
It's just a neuron with local connectivity

An activation map is a 28x28 sheet of neuron ouputs:
1. Each is connected to a small region in the input
2. All of them share parameters
5x5 filter -> 5x5 receptive field for each neuron

E.g with 5 filters,
CONV layer consists of neurons arranged in a 3D grid(28x28x5)

There will be 5 different neurons all looking at the same region
in the input volume.

Summary
- ConvNets stack CONV - POOL - FC layers
- Trend towards smaller filters and deeper architectures
- Trend towards getting rid of POOL/FC layers (just CONV)
- Typical architectures look like
[(CONV-RELU)*N-POOL?]*M-(FC-RELU)*K,SOFTMAX
where N is usually up to ~5, M is large, 0<=K<=2
but recent advacnes such as ResNet/GoogLeNet challenge this paradigm
